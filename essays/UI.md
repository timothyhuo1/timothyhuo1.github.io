## The Struggle 
When I first started with just HTML and CSS, I thought with just these skills, there are endless designs and possibilities that I could do. I started to go onto other popular websites and see what I could incorporate in my own website. Of course with just HTML and CSS, it is possible but it is a hassle to get what you want and envision. I started with watching boot camps on youtube that were two to three hours long. Even after watching these videos, I still had to watch tutorials on how to create a footer, displaying words over images, and more. The first time I created a website, it took me days to make it semi-professional. Each time I started a new section, there would error or something out of place and I would try every solution on youtube or websites such as W3Schools to solve them.  I ended up creating this website over and over again because I would learn a new way of doing something easier, such as formatting a navbar as a list. When I heard about UI frameworks, I didn’t think too much of it but once I started getting deeper into it, it was a huge game-changer for me. 

## My Experience With UI Frameworks 
The first UI framework that I tried to learn was bootstrap. I didn’t fully understand bootstrap, I just looked at examples on the bootstrap website and tried my best to incorporate them into my projects. I had to keep referring to the website, as I couldn’t get the hang of the names of the classes and notation. When I got into my software engineering class, we learned Semantic UI, which is another UI framework. For me, Semantic UI was much easier to get the hang of compared to bootstrap. The class names where labeled with common words that describe what they do, which made it very easy to follow. I wish I started learning UI frameworks sooner, there are many options and tools that UI frameworks provide that makes front-end web development simpler. For example, when I wrote my navbar with a dropdown feature in just HTML and CSS, it took me an hour just trying to get the dropdown functional. Compared to UI Frameworks such as Semantic UI, it only took me 10 minutes to get the same result. 

```
<div class="ui vertical menu">
  <div class="ui dropdown item">
    Categories
    <i class="dropdown icon"></i>
    <div class="menu">
      <a class="item">Electronics</a>
      <a class="item">Automotive</a>
      <a class="item">Home</a>
    </div>
  </div>
</div>

Retrieved from https://semantic-ui.com/collections/menu.html
```
Here is an example of a dropdown menu using Semantic UI. This example shows how organized the code is with UI frameworks. For this Semantic UI dropdown menu, all the classes are labeled and represent what they do. Another useful tool in Semantic UI is how you can reference icons. In the example, it calls the dropdown icon class which provides a dropdown icon to the navbar. It is very simple and you could adjust the size of the icon with just one word. If you want to icon large, you just type large in the quotations of the class. 

## Thoughts on UI Frameworks 
I would 100% recommend UI Frameworks to those who are familiar with HTML and CSS. Even though it takes a while to learn new Frameworks on top of HTML and CSS, it was worth it at the end. The more you practice using UI Frameworks, the more comfortable you would get and it is a tool that can be used in any projects relating to web development. It did take me a while to get the feel of Semantic UI, but they also provide a website with examples of what it should look like. I found it very useful that the Semantic UI website provides an example code on a class and what it looks like on a browser. There are many things that I never knew that was possible with UI Frameworks and I’m there is still so much to experiment with.
